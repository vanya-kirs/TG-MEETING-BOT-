Настройка интеграции с Google Таблицами

Этот бот включает автоматическую синхронизацию с Google Таблицами для управления данными тренеров и пользователей.

⸻

Структура Google Таблицы

Бот синхронизирует данные с таблицей Google, которая имеет следующую структуру:

Лист 1 — Основные данные пользователей

Автоматически синхронизируется при регистрации пользователя:
	•	Столбец A: ФИО — только для записи скриптом
	•	Столбец B: Номер телефона — только для записи скриптом
	•	Столбец C: Дата рождения — только для записи скриптом
	•	Столбец D: Дата начала работы — только ручное редактирование
	•	Столбец E: Спортивные и личные достижения — только ручное редактирование

⸻

Лист 2 — Образование и квалификация
	•	Столбец A: Имя тренера
	•	Столбец B: Даты повышения квалификации — синхронизируются из админ-панели бота
	•	Можно также редактировать вручную

⸻

Лист 3 — Медицинские записи
	•	Столбец A: Имя тренера
	•	Столбец B: Даты медицинской книжки — синхронизируются из админ-панели бота
	•	Можно также редактировать вручную

⸻

Лист 4 — Шкафчики
	•	Только ручное управление

⸻

Лист 5 — Формы
	•	Только ручное управление

⸻

Инструкция по настройке

1. Создай сервисный аккаунт Google
	1.	Перейди на Google Cloud Console￼
	2.	Создай новый проект или выбери существующий
	3.	Включи Google Sheets API и Google Drive API
	4.	Перейди в Credentials → Create Credentials → Service Account
	5.	Создай сервисный аккаунт и скачай JSON-файл ключа
	6.	Сохрани JSON-файл, например в папку credentials/service_account.json

⸻

2. Поделись таблицей с сервисным аккаунтом
	1.	Открой нужную Google Таблицу
	2.	Нажми «Поделиться»
	3.	Добавь e-mail сервисного аккаунта (указан в JSON-файле в поле client_email)
	4.	Дай ему права Редактора

⸻

3. Настрой переменные окружения

Добавь в свой файл .env:
# Настройки Google Sheets
GOOGLE_SHEETS_ID=your_spreadsheet_id_here
GOOGLE_SERVICE_ACCOUNT_PATH=/path/to/service_account.json

Как найти ID таблицы:
	•	Открой Google Таблицу
	•	Посмотри на URL:
https://docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit
	•	Скопируй часть SPREADSHEET_ID

⸻

4. Установи зависимости

Необходимые библиотеки уже указаны в req.txt:
pip install -r req.txt

Основные пакеты:
	•	gspread — библиотека для работы с Google Sheets API
	•	google-auth — библиотека для аутентификации в Google

⸻

Как это работает

Автоматическая синхронизация при регистрации пользователя

Когда новый пользователь завершает регистрацию, бот автоматически добавляет в Лист 1:
	•	ФИО
	•	Номер телефона
	•	Дату рождения

⸻

Автоматическая синхронизация при действиях администратора

Когда администратор добавляет данные через админ-панель:
	•	Даты медицинской книжки → записываются в Лист 3
	•	Даты квалификации → записываются в Лист 2

Бот ищет тренера по имени и:
	•	обновляет существующую строку, если тренер найден;
	•	создаёт новую строку, если тренер не найден.

⸻

Устранение неполадок

Бот запускается, но синхронизация не работает
	1.	Проверь наличие файла .env и правильность переменных
	2.	Убедись, что путь к JSON-файлу верный
	3.	Проверь, что e-mail сервисного аккаунта имеет права редактора в таблице
	4.	Посмотри логи — там будет текст ошибки

⸻

Предупреждение «GOOGLE_SERVICE_ACCOUNT_PATH not set»

Бот продолжит работать, но пропустит синхронизацию с таблицей.
Это нормально, если интеграция с Sheets не используется.

⸻

Ошибки прав доступа

Проверь:
	•	что сервисный аккаунт добавлен в таблицу с правами Редактора;
	•	что JSON-файл ключа корректный и не повреждён;
	•	что в Google Cloud включены Sheets API и Drive API.

⸻

Ограничения
	•	Бот не читает данные из таблицы обратно
	•	Автоматически синхронизируются только некоторые поля
	•	Остальные значения нужно редактировать вручную
	•	Если ФИО пользователя/тренера изменилось — обнови таблицу вручную
